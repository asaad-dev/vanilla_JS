L'ordre d'exécution du code par le navigateur

Par défaut, un navigateur va lire et exécuter le code dans l'ordre de son écriture

Lorsque le navigateur arrive script, il va stopper le traitementdu reste du HTML jusqu'à ce que le code JavaScript soit chargé dans la page et exécuté

Résumons ce qu'il se passe dans ce cas :

  1. Le navigateur commence à analyser (ou à traiter) le code HTML
  2. Le navigateur rencontre un élément script
  3. Le contenu JS est demandé et téléchargé (si il est en externe) puis exécuté. Durant tout ce temps l'analyseur du navigateur bloque l'affichage du HTML ce qui peu ralentir, si le script est long, l'affichage de la page
  4. Dés que le JS a été exécuté, le contenu HTML est affiché

Ce souci d'attente est résolu grâce au téléchargement asynchrone des donné avec l'attribus async ou defer dans le cas oû l'on fait appel à fichiers externes.

Le point virgule ; est utilisé pour signifeer la fin d'une instruction(une variable, une condition, une boucle etc.)
